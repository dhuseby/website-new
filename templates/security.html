Security

libp2p is still under active development and there may be problems with our protocols or errors in our implementations. We take security vulnerabilities very seriously and follow responsible disclosure best practices. If you discover a security issue, please bring it to our attention right away!

Reporting a Security Vulnerability

If you find a vulnerability that may affect live deployments—such as a remote execution exploit—please report it privately. DO NOT file a public issue. If you are unsure about a particular issue, please reach out to security@libp2p.io

If the issue is an implementation weakness that cannot be immediately exploited or is not yet deployed, plese file a public Github issue to get it resolved.
